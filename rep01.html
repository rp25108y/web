第1週目

1-1 サイエンスアート

1.内容
scratch を用いて、サイエンスアートを作成した。
ネコの動きに合わせて線を引き、その線の色を移動する毎に変更することで滑らかな色の変化を作った。
繰り返し移動させることで曲線の様な物を表現することが出来る。
2.感想
一回の動作でネコが移動する距離を大きくすると画面の端に到達してしまい、又小さくしても回転する角度が小さすぎても画面の端に到達してしまうので、その加減が難しかった。
100進んで200戻って100進むというコードは他の簡潔な表現が存在しても良いのにと感じた。

1-2 ゲーム

1.内容
ネコをキーボードの矢印キーで移動させ、落ちてくる矢印を獲得するゲームを作成した。
回転方向を左右のみとすることで、一般的な 2D ゲームと同じ様にネコが動く様になった。
2.感想
矢印キーで移動する際、方向転換をすぐに行なうことができず、又長押しだと認識するまでの時間が長く操作性が悪かった。
そのため、「ずっと」のブロック内で押されたかどうか判定する方が滑らかに動きそうだと感じた。

1-3 ホームページ作成

私のホームページ
1.内容
ホームページを作成するためのコードが格納されたリポジトリをフォークし、 github pages でそのコードを公開した。
html ファイルを変更することで、ブラウザで表示されるページに書かれている内容を変更した。
2.感想
gtihub pages を利用することで、簡単にホームページを作成、公開できる事は知っていたが、新しく github action によってホームページが公開されていたことを知ることが出来た。
yourID を自分のものに書き変えましょう、という指示が多く、外部ファイルに yourID を保存して、それを html で読み込む形にすれば、ドメインが変更されたときに対応しやすくなると感じた。
